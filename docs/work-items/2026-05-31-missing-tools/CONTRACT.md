# CONTRACT — Missing-tool plugin onboarding

This is the **shared contract** that two parallel subagents key off:
- **Subagent A** (mock data + acceptance docs) — produces the fixtures and the per-tool 接入/验收 docs.
- **Subagent B** (implementation) — writes the parser/plugin + unit tests that consume A's fixtures.

Both MUST agree on the exact filenames, formats, and expected normalized token tuples below. Do not deviate; if a change is needed, update this file first.

## Scope (decided)

In scope — local-scannable tools only:
| Tool | Source | On-disk path (canonical) | Form |
|---|---|---|---|
| **Qoder Desktop** | SQLite | `~/Library/Application Support/Qoder/SharedClientCache/cache/db/local.db` | **native** |
| **Kiro CLI** | SQLite | `~/Library/Application Support/kiro-cli/data.sqlite3` | **native** |
| **Kimi Code** | JSONL | `~/.kimi/sessions/**/wire.jsonl` | **declarative** |
| **Antigravity** | JSONL | `~/.gemini/antigravity/**/*.jsonl` (+ `~/Library/Application Support/Antigravity/**`) | **declarative** |

Out of scope (not local-feasible — document the *reason*, no mock/impl):
- **Cursor**: local `~/.cursor/projects/*.jsonl` carries NO token usage (`skipUsageProjection`); tokens only exist in the remote `cursor.com` CSV API and the shell-hook live capture — neither is a local file scan. → 接入文档说明"不支持本地采集"。
- **GitHub Copilot** (OTel JSONL): borderline; the OTel exporter is opt-in and absent by default. → document as "opt-in only", no mock this round.
- **Cursor global CSV / all shell-hook tools (Qwen/iFlow/Aone/Cline/…)**: require a daemon/HTTP-hook architecture TokenBar doesn't have. → out.

## Normalization model (TokenBar, already implemented)

`TokenNormalizer.normalize(input:output:cacheRead:cacheCreation:reasoning:inputIncludesCached:)`
- clamp each provider value to `>= 0`
- if `inputIncludesCached == true`: `cacheRead = min(cacheRead, input)`, then `input -= cacheRead`
- 6-tuple stored: `inputTokens / outputTokens / cacheReadTokens / cacheCreationTokens / reasoningTokens(=0 unless provided) / total`

Per-tool `inputIncludesCached` (from research doc):
- Qoder Desktop → **true**
- Kiro CLI → (conversation JSON; treat as **false** unless mock proves otherwise)
- Kimi Code → **false**
- Antigravity → **false**
